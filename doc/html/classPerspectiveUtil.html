Returns the point coordinate on the screen for the given parameters.

This method call suppose that the caller already knows the vanish points (those can be calculated from a previous call of perspectivePoint or calculateVanishPointsForAngle). This method is faster than the other since doesn't need to recalculate the vanish points. Use this if there are many points to estimate (i.e. map rendering).

Parameters
mapPointthe point representing the coordinate in the map (2D).
cameraPointthe camera point (observer) in the map (2D).
basePointthe baseline point on the screen.
cameraAnglethe angle of observation from the camera to the map point.
horizonYthe y-coordinate of the horizon line on the screen.
mapTileSizethe size that will be used for the tiles.
[in]vp1the vanish point on the horizon line opposite to the column point used for build the perspective point.
[in]vp2the vanish point on the horizon line opposite to the row point used for build the perspective point.
Returns
the point with screen coordinates in perspective view.
double PerspectiveUtil::scaleForPerspectivePoint ( const Point &  perspectivePoint,
const Point &  vanishPoint,
const Point &  constructionPoint 
)
static

Returns the scale that an object situated on the baseline (which "is in contact" with the screen) will have if situated on the given perspective point with the given vanish and construction point (which must have already been calculated).

If the perspective point is over the baseline (as it ought to be) the scale factor will be <= 1.0 (a further object is seen little).

Parameters
perspectivePointthe point on the screen perspective view.
vanishPointa vanish point used for build the perspective point.
constructionPointthe construction point opposite to the given vanish point.

the height of an object situated on the perspective point.
